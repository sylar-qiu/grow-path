# 第五章：客户沟通实战框架

前面四章是知识储备，这一章是**行动指南**——从第一次跟客户接触到项目落地，每一步该问什么、该注意什么。

---

## 一、第一次接触：信息收集

当你第一次接触一个潜在的流片客户，用这个清单把信息收全。

### 必问清单

```
技术规格
├─ 工艺节点？（多少 nm / μm？）
├─ 工艺类型？（CMOS / BCDE / RF / HV？）
├─ 芯片尺寸？（X × Y mm² —— 成本计算核心）
├─ 光罩层数？（多少层 Mask？）
├─ 封装形式？（QFN / BGA / SOP？还是只拿 Die？）

项目阶段
├─ 当前到什么进度了？（选型 / 设计 / GDS 快好了？）
├─ 计划什么时候 Tape-Out？（匹配 MPW 班次）
├─ 是 MPW 验证还是量产？（决定报价模式）

商业信息
├─ MPW 要多少颗？量产月需多少片？
├─ 有没有目标 Wafer Cost？
├─ 有没有指定的晶圆厂？（SMIC / 华虹？）
├─ PDK 拿到没有？（判断他过了哪道门槛）

应用场景
├─ 芯片用在什么产品上？（手机 / 汽车 / IoT / 电源）
├─ 目标市场是国内还是海外？
```

### 沟通话术示例

> "为了给您做一个初步的评估，我们几个关键信息需要对齐一下：芯片用的是哪种工艺和大概尺寸？目前项目到什么进度了，计划什么时候流片？首年的产量预期大概是多少？"

> "PDK 拿到了吗？如果还在等，我们可以帮忙加速一下流程。"

> "您这边是自己有封装资源，还是需要我们后期一并处理？这个报价差异挺大的。"

### 这些信息对你有用

- **工艺 + 尺寸** → 估算 NRE
- **产量** → 判断 MPW 还是 Full Mask
- **Tape-Out 时间** → 是否有 MPW 班次匹配
- **PDK 状态** → 你的第一个价值切入点（加速拿 PDK）
- **封装需求** → 报价包含不包含封测
- **指定晶圆厂** → 决定找 SMIC 还是华虹还是其他

## 二、报价阶段：报价模板

### 报价的结构

```
┌─────────────────────────────────────────────┐
│                                        报价单 │
│                                               │
│ 工艺：0.18μm BCDE                             │
│ 光罩层数：36 层                               │
│ 类型：MPW                                     │
│                                               │
│ ┌── 一次性费用（NRE） ──────────────────────┐ │
│ │ 光罩分摊费：$42,000                      │ │
│ │ 工程服务费：$1,200                       │ │
│ │ NRE 合计：$43,200                        │ │
│ └─────────────────────────────────────────┘ │
│                                               │
│ ┌── 晶圆费用 ─────────────────────────────┐ │
│ │ 单片 Wafer Cost：$2,500                 │ │
│ │ 数量：25 片                              │ │
│ │ 晶圆小计：$62,500                       │ │
│ └─────────────────────────────────────────┘ │
│                                               │
│ ┌── 封装测试（如需） ──────────────────────┐ │
│ │ 封装费：Die 数量的 QC 报价                │ │
│ │ FT测试费：每颗 $XX                       │ │
│ └─────────────────────────────────────────┘ │
│                                               │
│ 交期：JDV 确认后 45 天                         │
│ 报价有效期：15 天                              │
└─────────────────────────────────────────────┘
```

### 报价注意事项

1. **明确 Die 还是 Chip** — 在报价单上标注，避免歧义
2. **报价有效期** — 晶圆价格有波动，通常给 15 天有效期
3. **NRE 是否可退** — 不可退，这是行业惯例。如果客户问，直接说
4. **多处留白** — 给后期谈判留空间（工程服务费通常有一定浮动空间）

## 三、项目推进：各阶段的沟通要点

### 设计准备阶段

| 你的动作 | 话术 |
|---|---|
| 确认 PDK 状态 | "PDK 那边大概什么时候能拿到？如果流程上需要加速我可以帮忙跟进。" |
| 确认 FSR 填写 | "FSR 填了吗？要不要我们帮忙过一遍，确认参数跟设计意图一致？" |
| 对齐时间线 | "MPW 下个班次的截止日期是 X 月 X 日，我们往前推到 X 日完成 Tape-Out 才能赶上。" |

### 物理验证阶段

| 你的动作 | 话术 |
|---|---|
| 关注 DRC 进展 | "DRC 情况怎么样？有没有需要跟晶圆厂沟通 Waive 的问题？" |
| 控制 Retooling 风险 | "如果这个阶段发现问题要改，现在改比后面改便宜得多。JDV 窗口期一过，改一层光罩 = $5,800 + 等排期。" |
| 确认 Tape-Out 日期 | "Tape-Out 时间定下来了吗？我们好通知晶圆厂排 JDV 窗口。" |

### JDV 确认阶段（最关键的窗口）

| 你的动作 | 话术 |
|---|---|
| 提前提醒 | "JDV 窗口大概 3-5 个工作日，需要您这边安排人逐层确认。我会盯紧时间线。" |
| 进展跟踪 | "JDV 看完了吗？还有几层没确认？窗口期还剩 2 天。" |
| 决策推动 | "我理解您想再检查一遍，但如果过了窗口期不确认，晶圆厂默认推进。有问题的话现在提，光罩还没做，改起来代价最小。" |

### 制造与交付阶段

| 你的动作 | 话术 |
|---|---|
| 周期跟踪 | "晶圆预计 X 月 X 日出货，我会跟进物流和交货。" |
| 良率通报 | "这批良率 X%，在正常范围内。如果低于预期，我跟晶圆厂沟通一下看看能不能调参数。" |
| 交付确认 | "Die 已经准备好了，您这边是自取还是我们安排寄送？" |

## 四、风险信号识别

在跟客户沟通的各个阶段，有些信号一出现就要警觉：

### 前期风险信号

| 信号 | 解读 |
|---|---|
| 客户说不清楚芯片具体尺寸 | 他的设计还没定型，离 Tape-Out 还远，不要报确定价格 |
| 客户说"多少钱都行，先做再说" | 大概率预算没概念，后期容易产生价格纠纷 |
| 客户说"我觉得这个工艺应该行"而不是"我确认用这个工艺" | 选型还没定，你报价早了 |
| 客户说"PDK 正在申请"超过 2 个月 | 资质审核可能有障碍，需要你介入加速 |

### 执行阶段风险信号

| 信号 | 解读 |
|---|---|
| DRC 反复没过，而且客户态度变快 | 项目可能延期，准备调整排期预期 |
| 客户在 JDV 窗口期最后 1 天还在犹豫 | Retooling 风险极高，你需要强硬推动 |
| 客户突然说"我想改一下设计" + "已经 Tape-Out 了" | Retooling 不可避免，开始算额外费用 |
| 客户说"先把这一批做了，量产后再说" + 不提量产后怎么走 | 客户可能只做第一版就不做了，你的长期利润不成立 |

### 你能做的

- 信号刚出现时就提醒客户，而不是等出问题再补救
- 有些信号是沟通问题（客户没理解流程），可以教
- 有些信号是项目问题（设计不成熟、预算不清晰），需要客户自己解决，你注意不要替客户承担不属于你的风险

## 五、跟客户建立长期关系的沟通原则

### 原则 1：做"流程导航员"，不做"推销员"

你在客户面前的角色是帮他走通芯片制造这条路的人。只夸自己关系多好、价格多低没用，拿出对流程的熟悉度和专业的沟通姿态，你才能赢得信任。

### 原则 2：把技术语言翻译成商业语言

| 技术语言 | 商业语言 |
|---|---|
| "要 Retooling 5 层光罩" | "要追加 3 万美金，项目延后 6 周" |
| "DRC 有 200 个违规" | "有 200 个需要处理的小问题，但大部分可以跟晶圆厂豁免，我们安排一次电话对齐" |
| "MPW 班次错过了" | "等下一班的话，样品要晚 3 个月。或者我们走 Full Mask，快但贵一些" |

### 原则 3：预期管理比价格更重要

芯片行业最伤信任的不是价格高，而是**预期没对齐**。所有后期的不满意，都是因为前期没说清楚：
- 报价时说清 Die 还是 Chip
- MPW 启动时说清"MPW 后发现要量产可能需要重开光罩"
- JDV 窗口期说清"确认了就不能改"
- 交期说清"不包括节假日和晶圆厂检修"

把丑话说在前面，比事后补救好无数倍。
