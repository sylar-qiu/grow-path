# 第二章：芯片全生命周期流程（带行业执行惯例）

一颗芯片从想法到成品，经历 5 个阶段。每个阶段我都按"标准操作 → 行业惯例 → 实际博弈点"三层结构来写。

---

## 阶段一：设计与准备

### 标准操作

1. **定义规格** — 客户确定芯片的功能、性能、功耗、成本目标
2. **工艺选择** — 确定制造工艺节点（如 55nm、40nm、28nm、0.18μm 等）
3. **PDK 申请** — 向晶圆厂申请工艺设计工具包
4. **建 Code** — 在晶圆厂开通客户账户，获得下单资格

### 关键交付物

电路设计图、技术方案、工艺选型报告

### 行业惯例

**关于工艺选择：**
- 客户通常不是从零选工艺，而是基于已有设计经验或上一代产品的工艺延续。你不太需要帮客户决定用多少 nm，但需要能听懂他说的"0.18 BCDE"、"40nm LL"、"0.18um RF"分别对应什么场景
- 同一种工艺节点下有多种变体（详见第四章术语辨析），客户选错了等于后面全部白做。这是你在早期沟通中需要确认的关键信息

**关于 PDK 申请：**
- PDK 是晶圆厂的核心机密，100% 属于晶圆厂，每个工艺节点都有独立的 PDK
- 申请流程：客户提交资质 → 签署 NDA → 晶圆厂评估（通常 2-4 周）→ 发放 PDK
- **这里有个实际博弈：** 晶圆厂放 PDK 意味着"我承认你是一个正经的设计公司"，所以他们不会随便放。你作为渠道，可以帮客户加速这个流程——因为 SMIC 信任大渠道带来的客户，而不是来路不明的个人
- PDK 通常免费，但绑定必须通过该晶圆厂流片。如果客户拿 PDK 去其他厂流片，属于违约

**关于建 Code：**
- 这是中小 Fabless 最大的实际障碍
- **门槛：** SMIC 要求月产能承诺通常在千片以上，新客户开户 3-6 个月
- **绕过的实际做法：** 客户不自己建 Code，而是通过你或 Service Provider 的 Code 下单。你的 Code 背后是大客户的月产能流水，所以 SMIC 认
- **费用结构：** 建 Code 本身有一次性费用（约几千美金），但真正的成本是月产能承诺——如果你承诺了 1000 片/月但实际只下 100 片，晶圆厂会找你谈话
- **你作为渠道的优势：** 你手头有多个客户的总订单量来凑承诺，单个客户的波动不会触发预警

```
实际案例判断：
客户说"我刚成立的公司，想用 SMIC 55nm 流片"
→ 如果他走正规渠道：建 Code 6 个月，等不了
→ 如果走你：用你既有 Code，直接可以排 MPW 班次
这是你最有杀伤力的卖点之一
```

## 阶段二：物理设计与验证

### 标准操作

1. **前端设计** — 用硬件描述语言（Verilog/VHDL）编写代码，功能仿真
2. **后端设计** — 布局布线（Place & Route），将 RTL 代码转换成物理版图
3. **物理验证** — DRC（设计规则检查）+ LVS（版图与电路图一致性检查）
4. **Tape-Out** — 正式将 GDSII 文件提交给晶圆厂

### 关键交付物

GDSII 文件（芯片的最终制造图纸）

### 行业惯例

**这个阶段是整个流程中技术最复杂、最容易出问题的阶段。** 大部分涉及你业务的问题都出在这里。

**关于 DRC：**
- DRC 是晶圆厂设定的制造规则检查，确保你的版图能用他的机器造出来
- **实际常态：** 第一次 DRC 几乎不可能全过。大型芯片的 DRC 问题可能有几百上千个，这是正常的
- **关键概念：Waive（豁免）** — 某些 DRC 违规是设计团队明知故犯的（为了性能或面积优化），如果没有实质制造风险，晶圆厂的工程师可以"waive"掉（批准豁免）。**问题在于：** 晶圆厂的 Waive 权限有限，超过某种程度需要升级审批，或者直接拒绝
- 你在 DRC 阶段的角色：不是帮客户改设计（你不是工程师），而是**帮客户跟晶圆厂沟通**，搞清楚哪些可以 waive、哪些必须改、哪些可以争取时间

**关于 Retooling：**
- Retooling = Tape-Out 之后发现设计问题，修改 GDS 并重新制作光罩
- **这是整个项目中最昂贵的错误**，是你的业务中最重要的风险点
- 为什么会发生 Retooling：
  - DRC 问题在 Tape-Out 前没清干净
  - 最后时刻客户发现功能缺陷
  - FSR 填错了（如选了错误的电压选项）
- Retooling 的成本：不是只有光罩费，还有**时间成本**（重新排班次 = 延后 1-3 个月）
- **你的价值：** 在 GDS 到 JDV 之间这个"黄金窗口"，推动客户高效做决策、锁定修改范围。拖得越久，Retooling 的可能性越大

**关于 FSR（Foundry Service Request）—— 一个常被忽视但极其关键的环节：**

FSR 是客户在正式流片前向晶圆厂提交的**工艺选项和制造规范表格**，包含：
- 选择的工艺节点（如 0.18 BCDE）
- 工作电压（如 pure5V）
- 器件类型
- 金属层数
- 特殊模块（如 eFlash）
- 测试结构
- 可靠性要求

**实际执行中 FSR 的重要性：**
- FSR 是 JDV 生成的依据。FSR 填错了 → JDV 错了 → 光罩做错了 → 芯片不能工作
- 邮件中双方反复确认"pure5V"选项，就是因为这个选择决定了某些光罩层要不要做（如 NW/PW 层）
- **行业惯例：** FSR 通常由 Design Service（设计服务公司）帮你填，但最终由客户签字确认。你作为渠道，要确保客户理解签字的分量——签了字就默认你确认了所有技术参数
- FSR 确认后，改动就是修改费

```
FSR 填错的后果实例：
客户要 5V 芯片，但 FSR 选了 3.3V 选项
→ 光罩做出来全对，但造出来的芯片所有晶体管耐压不够
→ 整批报废，NRE + Wafer Cost 全部打水漂
→ 谁的责任？客户签了字，所以客户付钱
→ 你作为渠道能做的：帮客户多一重检查，避免这种低级但致命的错误
```

## 阶段三：制造准备与光罩制作

### 标准操作

1. **JDV 确认** — 晶圆厂对 GDS 数据进行"打包"，生成最终制造数据，发给客户确认
2. **Mask Making** — 根据确认后的数据制作光罩（每一层电路一张光罩）
3. **工程批流片** — 用做好的光罩试产少量晶圆，验证工艺

### 关键交付物

光罩（Mask）、工程批晶圆

### 行业惯例

**关于 JDV —— 整个流程中最关键的决策节点：**

- **JDV 是什么：** 晶圆厂把客户 GDS 数据"翻译"成制造数据的最后一步。翻译结果发给客户确认，就像印刷前看"打样"
- **JDV 确认后，数据进入光罩制作。光罩一做，任何修改都涉及重新制作光罩（Retooling）**
- **实际执行细节：**
  - JDV 确认的窗口期通常只有 3-5 个工作日
  - 客户需要在窗口期内逐层确认。36 层光罩就要看 36 层的 JDV 图
  - 窗口期过了不确认，晶圆厂默认"同意"，直接推进——这是行业的通用做法
  - 但如果你发现有问题，窗口期之后才提，晶圆厂的态度会**立即变硬**：要么付费修改，要么不接受修改
- **你的角色很关键：** 推动客户在窗口期内完成 JDV 确认。很多 Fabless 的老板在这阶段拖延——他们怕签字后有责任。你需要帮他明白：不签的代价更大
- **关于"完美主义陷阱"：** JDV 确认不是让客户逐像素检查设计是否有改进空间，而是确认"制造数据是否能正确还原设计意图"。有客户试图在 JDV 阶段改设计——这是死路

**关于 Mask Making（光罩制作）：**

- 光罩厂和晶圆厂通常是分开的（详见第四章）。顶尖 Foundry（台积电、英特尔）自有光罩厂，但大部分 Foundry 外购光罩
- 光罩制作的精度要求极高：0.18μm 工艺的光罩是芯片版图的 4-5 倍放大，但误差要求在纳米级别
- **光罩成本是 NRE 的大头。** 单层光罩的价格取决于工艺节点——先进工艺（如 28nm）单层光罩比成熟工艺（0.18μm）贵数倍
- 你之前提到的"36层mask 21万美金"就是个典型报价。平均每层约 5800 美金，在成熟工艺（如 0.18μm）属于正常范围

**关于光罩厂与晶圆厂的关系：**

- **行业常态：** 分开。独立光罩厂（Photronics、DNP 等）为多家晶圆厂服务
- **巨头模式：** 台积电、英特尔自建光罩厂，保证保密性和供应链安全
- **你的业务场景：** 通常不需要跟光罩厂直接打交道，流程通过晶圆厂走。你只需要理解光罩成本如何影响 NRE 报价

**关于工程批：**
- 光罩做好后，晶圆厂先跑几片工程批验证工艺参数
- 工程批的晶圆一般不给客户（用于调试工艺），但如果客户急用样品，可以协商拿工程批的 Die 做测试
- **行业惯例：** 工程批如果良率极低（比如 < 10%），晶圆厂可能会调整工艺参数重跑，这个调整**不一定**收费。但如果问题出在设计上（不是工艺参数），客户要承担延迟成本

## 阶段四：晶圆制造

### 标准操作

1. **正式流片** — 用确认后的光罩和工艺参数，在 Fab 中批量生产晶圆
2. **工艺控制** — 数百道工序（光刻、刻蚀、离子注入、化学机械抛光等），每道有严格参数

### 关键交付物

Wafer（晶圆），上面有数百到数千颗裸片（Die）

### 行业惯例

**关于生产周期：**
- 从流片到出晶圆：成熟工艺（0.18μm-55nm）约 30-45 天，先进工艺（28nm 以下）约 60-90 天
- 你提到的"1.2 天/层 vs 行业 2 天/层"——循环时间（Cycle Time）是按光罩层数算的，不是按整片晶圆。36 层的芯片，1.2 天/层 ≈ 43 天；2 天/层 ≈ 72 天
- **这个差异就是你的竞争力：** 能提供更短的交期，对时间敏感的项目是巨大吸引力

**关于产能（Capacity）：**
- Foundry 的产能分配是**季度性**的，每季度初确定本季度的产能配额
- 产能优先给大客户。小客户在产能紧张时可能被无限期推迟
- **你的渠道价值：** 你的 Code 背后有大客户的配额，即使你的客户是小的 Fabless，产能优先级也按大客户走

**关于良率（Yield）：**
- 晶圆上不是所有 Die 都工作。良率 = 合格 Die 数 / 总 Die 数
- 新工艺 + 新设计的良率通常不高（50-70%），成熟后可达 90%+
- **良率问题有几种可能：**
  - 工艺问题（晶圆厂的锅）— Foundry 会免费调参数
  - 设计问题（客户的锅）— 客户承担
  - 随机缺陷（谁都说不清）— 按行业惯例，良率低于某个阈值（如 50%）才触发赔偿机制
- **你需要知道的关键点：** 良率直接影响有效 Wafer Cost。如果一片晶圆 $2200，良率 70%，等于每片有效 $3143

## 阶段五：封装与测试

### 标准操作

1. **CP 测试**（Chip Probe）— 晶圆切割前，对每颗 Die 做初步电性测试，标记良品/不良品
2. **划片** — 将晶圆切割成单独的 Die
3. **封装** — 将 Die 封装为 Chip（QFN、BGA、SOP 等不同封装形式）
4. **FT 测试**（Final Test）— 对封装后的芯片做完整功能和性能测试

### 关键交付物

Chip（芯片成品）

### 行业惯例

**关于 Die vs Chip 交付（这是业务中最常见的沟通起点）：**

- **默认交付 Die（裸片）** — 行业惯例中，MPW 项目默认交付 Die，不包含封装测试。因为客户经常自己找封装厂
- **交付 Chip（成品芯片）** — 需要额外付封装费 + 测试费，合计约总成本的 20-30%
- **你的业务中：** 报价时一定要问清楚"客户要 Die 还是 Chip"。前期没确认，后期加封装费会让客户觉得你报价虚高

**关于封装厂：**
- 封装通常是独立于晶圆厂的环节（封装厂 ≠ Foundry）
- 客户可能有自己的封装合作方，也可能让你推荐
- 封装形式的选择直接影响芯片的最终应用场景

**关于测试：**
- CP 测试是晶圆厂的标配，FT 测试通常是封装厂或第三方测试厂做
- 测试费用按**每颗芯片**算，不是按每片晶圆算
- **行业惯例：** 小客户（MPW 几十颗）的测试费单价会比大批量量产芯片贵很多，因为测试厂要单独 setup

## 阶段六：量产

### 标准操作

MPW 验证通过后，客户决定进入量产。
- 评估采用 Full Mask（全掩模新开光罩）还是沿用 MPW 光罩
- 签订量产合同，确定 Wafer Cost 和月产能
- 按季度/月度排产，持续交付

### 行业惯例

**关于 MPW → Full Mask 的切换：**

- MPW 验证通过了，不等于可以直接量产。MPW 的光罩是"拼单"做的，设计可能不是最优——Full Mask 阶段会**重新优化光罩布局**，有可能产生额外 NRE
- **实际场景：** 客户 MPW 成功后满怀期待要做量产，结果 Full Mask 阶段发现 NRE 跟预期差不多——这是常见的沟通断裂点。你要提前帮客户理解这个预期

**关于量产合同：**
- 量产合同的 Wafer Cost 通常比 MPW 的报价低（量大价优）
- 但量产合同有**最低采购承诺**——比如每月最少 50 片，签一年
- 如果客户产品卖不好，低于承诺量，要么补差价，要么罚款
- **你在其中的角色：** 帮客户&晶圆厂双方协调一个合理的承诺量，不要太激进

**关于产能锁定：**
- 一旦进入量产，你可以在每季度初锁定下季度的产能配额
- 需要预测用量——准了皆大欢喜，不准了要么多付要么少做
- **行业惯例：** 预测的准确度影响客户在下个季度的优先级。预测永远偏低的客户（每次都要追加）会慢慢失去优先权

## 流程总结：你的角色在各阶段的实际价值

| 阶段 | 客户最焦虑的问题 | 你最有价值的动作 |
|---|---|---|
| 设计准备 | "PDK 拿不到、Code 建不了" | 用既有渠道帮客户绕过门槛，快速启动 |
| 物理验证 | "DRC 反复不过、Retooling 风险" | 帮客户跟晶圆厂沟通 Waive、控制 Retooling 范围 |
| 制造准备 | "JDV 窗口期不敢签字" | 推动确认、监控时间线 |
| 晶圆制造 | "产能够不够、交期长不长" | 用大客户配额保障产能和速度 |
| 封测 | "Die 还是 Chip？多少钱？" | 明确报价标准、推荐封装伙伴 |
| 量产 | "Wafer Cost 能不能再低？" | 谈长期价格、协调产能预测 |
