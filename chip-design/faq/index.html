芯片流片常见问题详解¶

基于实际业务中的常见问题，逐一解答如下。

1. 流片交付物：Die 还是 Chip？¶

行业常规： 流片后的交付物可以是 Die（裸片） 或 Chip（芯片，即封装测试后的成品），具体取决于项目阶段、客户需求及合同约定。

交付 Die 的常见情况¶

  • MPW（多项目晶圆）项目 — 最常见的模式。MPW 主要用于工程批验证、样品测试，客户需要拿到裸片进行自己内部的详细测试、可靠性实验，或者送至不同的封装厂进行封装工艺评估
  • 客户拥有自己的封装与测试能力或固定合作方
  • 降低成本 — 跳过封装测试环节，可以降低本次流片的直接费用

交付 Chip 的常见情况¶

  • 量产项目 — 特别是通过 Full Mask / NTO 方式进行的正式量产，通常直接交付封装测试好的成品芯片，以便客户用于终端生产
  • Turn-Key 项目 — 客户要求设计公司或服务商提供"交钥匙"方案，从流片到最终可销售的芯片全部完成
  • 小公司或项目 — 自身没有封装测试资源，倾向于由晶圆厂或服务商一并完成

费用关系¶

Die 到 Chip 之间，需要增加封装费和测试费。在之前的沟通中提到的报价（如"50 颗 18k 美金"）通常默认为 Die 的费用。如果需要成品 Chip，必须在报价阶段明确，因为封装测试费可能占总成本的 20%-30%，是一笔不小的额外开销。

关键行动： 在项目开始时就必须与客户明确交付标准。

2. MPW 项目的发起模式¶

MPW 就像晶圆厂定期发出的"班车"：

发起方¶

由晶圆厂主动、定期发起。晶圆厂会提前（如按季度或年度）公布其不同工艺平台的 MPW 运行计划（Schedule），即"班次表"。

运作模式¶

  1. 班次发布 — 晶圆厂公布未来一段时间内各工艺节点的 MPW 运行时间
  2. 席位预订 — 设计公司（客户）根据自身项目进度，向晶圆厂或其代理申请预订某个班次的"席位"（即晶圆上的部分面积）
  3. 数据提交 — 在班次截止日期前，提交最终的 GDS 数据
  4. "发车"制造 — 席位预订满或截止日期到，该班次晶圆即投入生产

结论¶

这是一个晶圆厂为降低小客户流片门槛、提高其产线利用率而设立的标准化、周期性的服务。客户是"乘客"，需要主动"购票上车"。

3. 晶圆厂与光罩厂的关系¶

常规情况¶

在半导体产业链中，晶圆厂和光罩厂通常是分开的专业工厂。光罩制造是芯片制造的前道工序，需要极高的精度和洁净度，是一门独立且高技术的生意。

存在形式¶

形式 代表 特点
独立光罩厂 Photronics（美国）、DNP（日本）等 为多家晶圆厂和设计公司服务
晶圆厂自有光罩部门 台积电、英特尔 保证核心工艺的保密性和产能

总结¶

两者分开是普遍情况，但顶尖的集成制造巨头（IDM 或大型 Foundry）往往自有或控股光罩厂以保障供应链安全。如果某个渠道方"对中兴而言他都有"，可能指其集团内部具备完整链条，但这在行业中并非所有公司都能做到。

4. 关于 0.18 BCDE 5×5 中的 BCDE¶

"BCDE"是晶圆厂对其 0.18 微米工艺平台中一个特定"工艺套件"或"技术模块组合"的代号。

常见解读¶

在模拟/功率工艺中，它可能代表集成了多种器件类型的技术：

缩写 全称 用途
B Bipolar（双极型晶体管） 模拟信号处理
C CMOS（互补金属氧化物半导体） 数字电路基础
D DMOS（双扩散金属氧化物半导体） 高压功率器件
E eNVM / EPROM 等 嵌入式非易失性存储器

业务含义¶

客户选择"0.18 BCDE"，意味着他需要的不是最基础的 CMOS 数字工艺，而是一个能够同时制作模拟、数字、高压功率甚至存储单元的综合特色工艺，常用于电源管理、显示驱动等芯片。

5×5 则是芯片的尺寸（5mm × 5mm）。

5. 关于 FSR（Foundry Service Request）¶

在邮件往来中经常出现。FSR 的全称是 Foundry Service Request 或 Fab Service Request，中文可译为"流片服务请求表"。

是什么¶

这是客户在正式流片前，向晶圆厂提交的一份详细的工艺选项和制造规范表格。

填写内容¶

包括选择的工艺节点（如 0.18 BCDE）、电压、器件类型、金属层数、特殊模块（如 eFlash）、测试结构、可靠性要求等所有制造参数。

重要性¶

FSR 是制造合同的技术部分，是 JDV 生成的依据。 邮件中双方反复确认"pure5V"选项，就是为了确保 FSR 填写正确，从而生成正确的光罩层（避免不需要的 NW/PW 层），这直接关系到芯片功能和制造成本。

6. 关于 0.18μm RF 及同级别工艺类型¶

RF 是什么¶

Radio Frequency，指处理高频无线信号的射频工艺。它在基础 CMOS 工艺上进行了优化，以提供高性能的无源器件（电感、电容）、低噪声的晶体管和精确的模型，用于制造蓝牙、Wi-Fi、手机射频等芯片的无线部分。

同级别工艺类型¶

在同一工艺节点（如 0.18μm）下，晶圆厂会开发多种衍生的特色工艺以满足不同应用：

工艺类型 应用场景
混合信号工艺 优化模拟和数字电路的结合
高压工艺 支持几十伏甚至上百伏的工作电压，用于显示驱动、电源管理
嵌入式存储器工艺 集成 Flash、EEPROM 等存储单元
BCD 工艺 集成 Bipolar、CMOS、DMOS（见第 4 问）

目标工艺¶

客户根据其芯片的核心功能（是做无线连接、电源控制、还是 MCU）来选择对应的特色工艺。客户询问 0.18μm RF PDK，意味着他正在设计一款包含射频功能的芯片。

7. PDK 与 IP 的深度解析¶

这是一个非常核心且容易混淆的问题。

PDK 是什么¶

工艺设计工具包（Process Design Kit）。是晶圆厂提供给设计者的"设计规则和基础元器件库"。包含：

  • 设计规则文件（DRC / LVS）
  • 晶体管、电阻、电容等基础器件的 SPICE 模型和版图符号
  • 用于 EDA 工具的工艺库文件

IP 是什么¶

知识产权核（Intellectual Property）。是基于 PDK 设计好的、可复用的、具有特定复杂功能的模块，如 CPU 核（ARM）、USB 控制器、DSP 等。

比喻理解¶

PDK IP
比喻 砖瓦水泥和建筑规范 预制好的厨房或楼梯
产权 100% 属于晶圆厂，核心机密 属于 IP 设计公司（如 ARM）
获取方式 申请审核 + 签署 NDA 商务谈判 + 授权费 + 版税

PDK 的产权与申请¶

  • 产权 100% 属于晶圆厂，是其核心机密
  • 每个晶圆厂的 PDK 都不同，甚至同一厂的不同工艺节点也不同
  • 申请 PDK 不仅是管理举措，更是严格的法律和技术评估流程。晶圆厂需要：
  • 评估客户资质 — 防止技术泄露给竞争对手或不靠谱的团队
  • 签署 NDA — 绑定法律责任
  • 技术对接 — 确保客户能正确使用
  • 它不是"做生意时必然提供的基础东西"，而是在确认合作意向、评估项目可行性后才会提供的核心生产资料。申请是开启合作的第一步，而非最后一步

IP 申请的难度¶

IP 申请比 PDK 申请更难，原因在于：

  1. 商务模式复杂 — PDK 通常免费（绑定流片），而 IP 涉及高昂的授权费和持续的版税，谈判周期长
  2. 法律条款繁琐 — 授权协议涉及使用范围、生产地点、转让限制等复杂条款
  3. 技术评估深入 — 需要深度评估 IP 与客户设计的集成度和性能

总结¶

PDK IP
本质 "开干"的许可证和工具箱 昂贵的"高级预制件"
申请难点 资质审核和合规 商业谈判和复杂授权

8. 与客户沟通时需要了解的核心信息¶

必须主动询问。 这是评估项目、提供报价、管理风险的基础。

需要了解的内容清单¶

技术规格¶

问题 为什么重要
工艺节点与类型？（多少 nm？什么工艺？如 40nm LL, 0.18μm RF） 决定适用哪家晶圆厂、哪种工艺平台
芯片尺寸？（X × Y mm²） 直接决定一片晶圆能切出多少颗芯片，是成本计算的核心
层数？（大约多少层 Mask？） 与 NRE 直接相关
封装形式？（QFN、BGA 等） 关系封装费

项目阶段与计划¶

问题 为什么重要
当前阶段？（在选型、在设计、还是 GDS 已准备好？） 判断项目成熟度，你还需要投入多少精力
计划何时 Tape-Out？ 用于匹配 MPW 班次
项目性质？（MPW 验证还是 Full Mask 量产？） 决定报价模式

商业与产量¶

问题 为什么重要
MPW 要多少颗？量产首年/峰值月需求是多少？ 决定采用 MPW 还是 Full Mask，以及评估能否建 Code 的关键
有无明确的芯片目标成本？ 判断你的报价是否有竞争力

应用场景¶

问题 为什么重要
芯片用于什么产品？（手机、汽车、IoT？） 关系到对工艺可靠性、温度范围等的特殊要求

供应链偏好¶

问题 为什么重要
是否有指定的晶圆厂？（如 SMIC、华虹？） 决定你需要调动哪边的资源

沟通话术示例¶

"为了给您做一个初步的评估，我们需要了解几个关键信息：咱们芯片用的是哪种工艺和大概尺寸？目前项目进度如何，计划什么时候流片？以及首年的产量预期大概是多少？"

掌握了这些，你就能给出专业、有针对性的方案。
